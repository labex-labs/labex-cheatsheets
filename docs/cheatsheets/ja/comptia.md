## Security+ のコアコンセプト

### セキュリティの基礎

CIA トライアドと基本的なセキュリティ原則。

```text
# CIA トライアド
機密性 (Confidentiality): データのプライバシーと暗号化
完全性 (Integrity): データの正確性と真正性
可用性 (Availability): システムの稼働時間とアクセス可能性

# 認証の要素
知っているもの：パスワード、PIN
持っているもの：トークン、スマートカード
あなた自身であるもの：生体認証
あなたがすること：行動パターン
あなたがどこにいるか：位置情報
```

### 脅威の状況

一般的な攻撃と脅威アクター。

```text
# 攻撃の種類
- フィッシングとソーシャルエンジニアリング
- マルウェア (ウイルス、トロイの木馬、ランサムウェア)
- DDoS および DoS 攻撃
- 中間者攻撃 (Man-in-the-middle attacks)
- SQL インジェクションと XSS
- ゼロデイエクスプロイト

# 脅威アクター
- スクリプトキディ
- ハクティビスト
- 組織犯罪
- 国家主体アクター
- 内部脅威
```

### 暗号化

暗号化方式と鍵管理。

```text
# 暗号化の種類
対称暗号：AES, 3DES (同じ鍵を使用)
非対称暗号：RSA, ECC (鍵ペアを使用)
ハッシュ化：SHA-256, MD5 (一方向)
デジタル署名：否認防止

# 鍵管理
- 鍵の生成と配布
- 鍵のエスクローと復旧
- 認証局 (CA)
- 公開鍵基盤 (PKI)
```

### アクセス制御

ID 管理と認可モデル。

```text
# アクセス制御モデル
DAC: 任意アクセス制御
MAC: 強制アクセス制御
RBAC: ロールベースアクセス制御
ABAC: 属性ベースアクセス制御

# ID 管理
- シングルサインオン (SSO)
- 多要素認証 (MFA)
- LDAP および Active Directory
- フェデレーションと SAML
```

## トラブルシューティングの方法論

### CompTIA トラブルシューティング手順

技術的な問題解決のための標準的な方法論。

```text
# 6 ステッププロセス
1. 問題の特定
   - 情報の収集
   - ユーザーへの症状に関する質問
   - システムへの変更点の特定
   - 可能な場合は問題の再現

2. 推定原因の理論の確立
   - 明らかなことから疑う
   - 複数のアプローチを検討する
   - 簡単な解決策から始める

3. 原因を特定するための理論のテスト
   - 理論が確認されたら、続行
   - 確認されなかった場合は、新しい理論を確立
   - 必要に応じてエスカレーション
```

### 実装とドキュメント化

トラブルシューティングプロセスの最終ステップ。

```text
# 残りのステップ
4. 実行計画の確立
   - 解決のためのステップを決定
   - 潜在的な影響を特定
   - 解決策を実装するか、エスカレーションする

5. 解決策の実装またはエスカレーション
   - 適切な修正を適用
   - 解決策を徹底的にテスト
   - 完全な機能を確認

6. 調査結果、アクション、および結果の文書化
   - チケットシステムを更新
   - 学んだ教訓を共有
   - 将来の発生を防ぐ
```
